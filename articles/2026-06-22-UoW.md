# UoW 工作单元模式
Unit of Work（简称 UoW，工作单元模式）是经典的企业级数据访问设计模式。

它的核心思想是：**维护一个业务事务内所有被修改的业务对象列表，跟踪对象的增、删、改状态，在业务流程结束时，将所有变更一次性批量提交到数据库；若中途出错，则所有变更整体回滚**，保证多步数据操作的原子性与数据一致性。

其核心特征包括：

-   **统一事务边界**：把多个零散的数据操作收拢到同一个逻辑事务中
-   **延迟写入策略**：先在内存跟踪变更，提交时才真正访问数据库
-   **自动变更跟踪**：自动识别实体的新增 / 修改 / 删除状态，生成对应 SQL
-   **依赖顺序执行**：自动处理表间外键依赖，按正确顺序执行写入操作

假设业务需求：交易下单时，步骤如下：

 1. 检查资金是否足够
 2. 冻结资金
 3. 生成订单
 4. 生成资金流水

先用最常见的 @Transactional + MyBatis 写法实现需求，核心代码：
```
import org.springframework.stereotype.Service;
import org.springframework.transaction.annotation.Transactional;
import javax.annotation.Resource;
import java.math.BigDecimal;
import java.util.UUID;

@Service
public class OrderService {

    @Resource
    private UserAccountMapper userAccountMapper;
    @Resource
    private TradeOrderMapper tradeOrderMapper;
    @Resource
    private AccountRecordMapper accountRecordMapper;

    /**
     * 下单：校验余额 → 冻结资金 → 创建订单 → 生成流水
     * 典型 @Transactional + MyBatis 即时执行模式
     * 每调用一次mapper立即执行一条SQL，全部在同一个事务内
     */
    @Transactional(rollbackFor = Exception.class)
    public void createOrder(Long userId, BigDecimal payAmount) {
        // ========== 步骤1：查询并锁定账户，校验余额是否充足 ==========
        // FOR UPDATE 当前读，加行锁，防止超卖
        UserAccount account = userAccountMapper.selectByUserIdForUpdate(userId);
        if (account == null) {
            throw new RuntimeException("账户不存在");
        }
        if (account.getAvailableBalance().compareTo(payAmount) < 0) {
            throw new RuntimeException("可用余额不足");
        }

        // ========== 步骤2：冻结余额（调用Mapper立即执行UPDATE SQL） ==========
        // 执行 update，SQL 立刻发给数据库执行，只是未提交
        int affect = userAccountMapper.freezeBalance(userId, payAmount);
        if (affect != 1) {
            throw new RuntimeException("资金冻结失败");
        }
        // 计算冻结后可用余额
        BigDecimal newAvailable = account.getAvailableBalance().subtract(payAmount);

        // ========== 步骤3：生成订单（INSERT 立即执行） ==========
        String orderNo = UUID.randomUUID().toString().replace("-", "");
        TradeOrder order = new TradeOrder();
        order.setOrderNo(orderNo);
        order.setUserId(userId);
        order.setTotalAmount(payAmount);
        order.setStatus(0); // 待支付
        tradeOrderMapper.insert(order);

        // ========== 步骤4：生成资金流水（INSERT 立即执行） ==========
        AccountRecord record = new AccountRecord();
        record.setUserId(userId);
        record.setOrderNo(orderNo);
        record.setChangeAmount(payAmount);
        record.setType(1); // 1=冻结
        record.setBalanceAfter(newAvailable);
        accountRecordMapper.insert(record);

        // 方法正常结束：AOP 统一 commit
        // 任意步骤抛异常：整体 rollback，前面所有已执行SQL全部回滚
    }
}
```
执行流程：

1.  进入方法，`@Transactional` AOP 拦截，获取连接 `setAutoCommit=false`，开启事务
2.  `selectByUserIdForUpdate` → **立即发送 SELECT SQL 执行**
3.  `freezeBalance` → **立即发送 UPDATE SQL 执行**
4.  `tradeOrderMapper.insert` → **立即发送 INSERT SQL 执行**
5.  `accountRecordMapper.insert` → **立即发送 INSERT SQL 执行**
6.  无异常：事务 `commit`，数据永久落库
7.  中途抛异常：整体 `rollback`，前面所有已执行 SQL 全部撤销

再用 UoW 模式实现需求，代码稍微复杂：
```
// 数据变更类型
public enum ChangeType {INSERT, UPDATE, DELETE}

// 变更条目（记录待执行操作）
@Data
@AllArgsConstructor
public class ChangeEntry {
    // 操作对象
    private Object entity;
    // 增/改/删
    private ChangeType changeType;
    // 执行目标：mysql / mq / es
    private String target;
}

// 持久化处理器接口（多渠道扩展核心）
public interface PersistenceHandler {
	  // 当前处理器匹配的目标标识
    String getTarget();
    // 批量执行一批变更
    void handle(List<ChangeEntry> entries) throws Exception;
}

// MySQL 持久化处理器
Component
public class MysqlPersistenceHandler implements PersistenceHandler {

    public static final String TARGET_MYSQL = "mysql";

    @Resource
    private UserAccountMapper accountMapper;
    @Resource
    private TradeOrderMapper orderMapper;
    @Resource
    private AccountRecordMapper recordMapper;

    @Override
    public String getTarget() {
        return TARGET_MYSQL;
    }

    @Override
    public void handle(List<ChangeEntry> entries) {
        for (ChangeEntry entry : entries) {
            Object entity = entry.getEntity();
            ChangeType type = entry.getChangeType();

            if (entity instanceof UserAccount) {
                UserAccount acc = (UserAccount) entity;
                switch (type) {
                    case UPDATE:
                        accountMapper.updateAccount(acc);
                        break;
                    case INSERT:
                        accountMapper.insert(acc);
                        break;
                    case DELETE:
                        accountMapper.deleteById(acc.getId());
                        break;
                }
            } else if (entity instanceof TradeOrder) {
                TradeOrder order = (TradeOrder) entity;
                if (type == ChangeType.INSERT) {
                    orderMapper.insert(order);
                }
            } else if (entity instanceof AccountRecord) {
                AccountRecord record = (AccountRecord) entity;
                if (type == ChangeType.INSERT) {
                    recordMapper.insert(record);
                }
            }
        }
    }
}

// 事务协调器
public interface TransactionCoordinator {
    /**
     * 执行一批变更，自动控制事务：全部成功则提交，任意异常全部回滚
     */
    void coordinate(List<ChangeEntry> allChanges, List<PersistenceHandler> allHandlers) throws Exception;
}

// MySQL 事务协调器
@Component
public class MysqlTransactionCoordinator implements TransactionCoordinator {

    @Resource
    private TransactionTemplate transactionTemplate;

    @Override
    public void coordinate(List<ChangeEntry> allChanges, List<PersistenceHandler> allHandlers) {
        // 构建 target -> handler 映射
        Map<String, PersistenceHandler> handlerMap = allHandlers.stream()
                .collect(Collectors.toMap(PersistenceHandler::getTarget, h -> h));

        transactionTemplate.execute(new TransactionCallbackWithoutResult() {
            @Override
            protected void doInTransactionWithoutResult(TransactionStatus status) {
                try {
                    // 按目标分组执行
                    Map<String, List<ChangeEntry>> group = allChanges.stream()
                            .collect(Collectors.groupingBy(ChangeEntry::getTarget));

                    for (Map.Entry<String, List<ChangeEntry>> entry : group.entrySet()) {
                        PersistenceHandler handler = handlerMap.get(entry.getKey());
                        if (handler == null) {
                            throw new RuntimeException("未找到持久处理器:" + entry.getKey());
                        }
                        handler.handle(entry.getValue());
                    }
                } catch (Exception e) {
                    status.setRollbackOnly();
                    throw new RuntimeException("MySQL事务提交失败", e);
                }
            }
        });
    }
}

// 非事务协调器，例如 redis，mq，Raft
@Component
public class NoTransactionCoordinator implements TransactionCoordinator {
    @Override
    public void coordinate(List<ChangeEntry> allChanges, List<PersistenceHandler> allHandlers) throws Exception {
        Map<String, PersistenceHandler> handlerMap = allHandlers.stream()
                .collect(Collectors.toMap(PersistenceHandler::getTarget, h -> h));

        Map<String, List<ChangeEntry>> group = allChanges.stream()
                .collect(Collectors.groupingBy(ChangeEntry::getTarget));

        for (Map.Entry<String, List<ChangeEntry>> entry : group.entrySet()) {
            PersistenceHandler handler handlerMap.get(entry.getKey());
            handler.handle(entry.getValue());
        }
    }
}

// 对外统一接口
public class CustomUnitOfWork {
    private final ThreadLocal<List<ChangeEntry>> changeHolder = ThreadLocal.withInitial(ArrayList::new);

    // 注册变更，只存入内存，无任何执行、无事务
    public void registerInsert(Object entity, String target) {
        changeHolder.get().add(new ChangeEntry(entity, ChangeType.INSERT, target));
    }

    public void registerUpdate(Object entity, String target) {
        changeHolder.get().add(new ChangeEntry(entity, ChangeType.UPDATE, target));
    }

    public void registerDelete(Object entity, String target) {
        changeHolder.get().add(new ChangeEntry(entity, ChangeType.DELETE, target));
    }

    // 获取当前所有变更
    public List<ChangeEntry> getPendingChanges() {
        return new ArrayList<>(changeHolder.get());
    }

    // 清空当前线程变更
    public void clear() {
        changeHolder.remove();
    }

    /**
     * 对外统一提交入口：外部传入事务协调器 + 所有处理器
     * UoW 本身不再绑定任何事务实现
     */
    public void commit(TransactionCoordinator coordinator, List<PersistenceHandler> handlers) throws Exception {
        List<ChangeEntry> changes = getPendingChanges();
        if (changes.isEmpty()) {
            clear();
            return;
        }
        try {
            // 交给协调器去控制事务+执行
            coordinator.coordinate(changes, handlers);
        } finally {
            clear();
        }
    }
}
```

执行逻辑：
```
@Service
public class OrderUowService {

    @Resource
    private CustomUnitOfWork uow;

    // 可注入不同协调器，按需选择
    @Resource
    private MysqlTransactionCoordinator mysqlCoordinator;
    
    @Resource
    private List<PersistenceHandler> allPersistenceHandlers;
    @Resource
    private UserAccountMapper accountMapper;

    public void createOrderWithMysqlTx(Long userId, BigDecimal payAmount) throws Exception {
        // 1. 资金检查
        UserAccount account = accountMapper.selectByUserIdForUpdate(userId);
        if (account.getAvailableBalance().compareTo(payAmount) < 0) {
            throw new RuntimeException("余额不足");
        }
        BigDecimal newAvail = account.getAvailableBalance().subtract(payAmount);
        account.setAvailableBalance(newAvail);
        account.setFreezeBalance(account.getFreezeBalance().add(payAmount));
        // 2. 更新资金（只是加入队列，延迟执行）
        uow.registerUpdate(account, MysqlReflectPersistenceHandler.TARGET_MYSQL);
        
        // 3. 保存订单（只是加入队列，延迟执行）
        String orderNo = UUID.randomUUID().replace("-", "");
        TradeOrder order = new TradeOrder();
        order.setOrderNo(orderNo);
        order.setUserId(userId);
        order.setTotalAmount(payAmount);
        order.setStatus(0);
        uow.registerInsert(order, MysqlReflectPersistenceHandler.TARGET_MYSQL);
         
        // 4. 生成资金流水（只是加入队列，延迟执行）
        AccountRecord record = new AccountRecord();
        record.setUserId(userId);
        record.setOrderNo(orderNo);
        record.setChangeAmount(payAmount);
        record.setType(1);
        record.setBalanceAfter(newAvail);
        uow.registerInsert(record, MysqlReflectPersistenceHandler.TARGET_MYSQL);

        // 5. 手动选择【MySQL事务协调器】提交
        uow.commit(mysqlCoordinator, allPersistenceHandlers);
    }

    // 切换无事务模式（比如只同步ES、Redis场景）
    public void syncEsData() throws Exception {
        // ... 注册变更
        uow.commit(noTxCoordinator, allPersistenceHandlers);
    }
}
```

执行流程：

1. 开启方法 → UoW创建空变更列表
2. 查询账户SQL（立即执行，校验余额）
3. 修改账户内存属性 → uow.registerUpdate(账户,mysql)、registerUpdate(账户,mq)
4. 构建订单、流水对象 → uow.registerInsert(订单,mysql)、registerInsert(流水,mysql)
5. 调用 uow.commit(MySQL事务协调器,所有处理器)
6. UoW清空线程变量

这种实现体现了 UoW 的优势：

 1. **写 SQL 延迟批量执行，缩短锁持有时间**
 2. 统一收集数据变更语句，实现**定制化开发**，例如可以选择写 db，mq，Raft 等中间件；

限于篇幅，本文没有展开介绍 UoW 的**内存变更追踪**，其有以下特点：

1.  开发层面：**防止漏更新、误更新**，降低编码出错率；
2.  SQL 层面：**合并多次更新、剔除无效更新、按需更新字段**，优化数据库性能；
3.  架构层面：**业务修改与持久化解耦**，面向对象编程更纯粹；
4.  治理层面：**统一获取变更前后数据**，方便全局变更审计；
5.  UoW 设计层面：补齐延迟写入闭环，真正实现「修改在内存，提交再落库」的工作单元思想。

UoW vs @Transactional + MyBatis 选型指南
-   **简单 CRUD、SQL 精细化调优、团队熟悉 MyBatis、存量 MyBatis 项目 → 选 `@Transactional + MyBatis`**
-   **复杂领域模型、多实体联动修改、追求面向对象开发、需要统一变更审计、想缩短事务锁时长 → 选 UoW 模式**
